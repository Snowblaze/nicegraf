{\footnotesize\ttfamily const void$\ast$ ngf\+\_\+shader\+\_\+stage\+\_\+info\+::content}

This shall be a pointer to a memory buffer containing the code for the shader stage.

The specific contents of the buffer depend on which backend nicegraf is being used with\+:
\begin{DoxyItemize}
\item for the Vulkan backend, nicegraf expects the SPIR-\/V bytecode for the shader stage.
\item for the Metal backend, nicegraf expects the source code for the shader stage in the Metal Shading Language.
\end{DoxyItemize}

Additionally, the Metal backend expects the code to contain a special comment, mapping all \texorpdfstring{$<$}{<}descriptor set, binding\texorpdfstring{$>$}{>} pairs to the native Metal argument table slots. The comment shall be a C-\/style block comment -\/ beginning with a forward slash, followed by an asterisk -\/ containing the following word\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{NGF\_NATIVE\_BINDING\_MAP}

\end{DoxyCode}


followed by a newline character.

Each of the following lines until the end of the comment shall have the following format\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{(s b) : m}

\end{DoxyCode}


where {\ttfamily s} is the set number, {\ttfamily b} is the binding number within the set, and {\ttfamily m} is the index of the corresponding resource in Metal\textquotesingle{}s argument table.

For example, let\textquotesingle{}s say the Metal shader refers to index 3 in the texture argument table. Adding the following line to the binding map comment


\begin{DoxyCode}{0}
\DoxyCodeLine{(0 1) : 3}

\end{DoxyCode}


would tell the nicegraf metal backend to use the third slot of the texture argument table when an image is bound to set 0, binding 1 using ngf\+\_\+cmd\+\_\+bind\+\_\+resources.

When compiling HLSL shaders using nicegraf-\/shaderc, the comment with the binding map is generated automatically. \mbox{\Hypertarget{structngf__shader__stage__info_ab9ea958a2085f765ab8ad5b7bf54ac02}\label{structngf__shader__stage__info_ab9ea958a2085f765ab8ad5b7bf54ac02}}
