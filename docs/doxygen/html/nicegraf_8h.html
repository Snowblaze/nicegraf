◆ NGF_DEVICE_LIMIT_UNKNOWN

#define NGF_DEVICE_LIMIT_UNKNOWN   (~0u)

This is a special value used within the ngf_device_capabilities structure to indicate that a limit value (i.e. max texture size) is not known or not relevant for the current backend.

◆ ngf_vertex_input_rate

enum ngf_vertex_input_rate
Enumerator
NGF_INPUT_RATE_VERTEX 

Attributes are read per-vertex. With this vertex input rate, each vertex receives its own set of attributes.

NGF_INPUT_RATE_INSTANCE 

Attributes are read per-instance. With this vertex input rate, all vertices within the same instance share the same attribute values.

Map a region of a given buffer to host memory. It is an error to bind a mapped buffer using any command. ◆ ngf_default_render_target()

ngf_render_target ngf_default_render_target ( )

Obtain a render target associated with the the current context's swapchain. If the current context does not have a swapchain, the result shall be a null pointer. Otherwise, it shall be a render target that has a color attachment associated with the context's swapchain. If the swapchain was created with an accompanying depth buffer, the render target shall have an attachment for that as well.

The caller should not attempt to destroy the returned render target. It shall be destroyed automatically, together with the parent context.
